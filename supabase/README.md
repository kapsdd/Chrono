# Настройка базы данных CHRONO (Supabase)

Вся схема собрана в один файл — **`setup.sql`**. Запускать миграции по
отдельности больше не нужно.

## Что сделать (один раз)

1. Открой проект в [Supabase](https://supabase.com) → раздел **SQL Editor**.
2. Нажми **New query**.
3. Открой файл `supabase/setup.sql`, скопируй **всё содержимое** и вставь в
   редактор.
4. Нажми **Run** (или Ctrl+Enter).

Готово. Файл идемпотентный — его можно запускать сколько угодно раз, данные не
теряются и ничего не ломается. Если раньше уже запускал `schema.sql` и
миграции — просто прогони `setup.sql` поверх, он добавит недостающее.

## Что должно быть включено в Supabase

- **Auth → Providers → Discord** — включён, с Client ID / Secret из Discord
  Developer Portal.
- **Auth → URL Configuration → Redirect URLs** — добавь:
  ```
  http://127.0.0.1:53117/auth/callback
  ```
  (для входа из desktop-приложения).
- **Database → Replication** — Realtime для таблиц `tasks`, `projects`,
  `project_members` включается автоматически последним блоком `setup.sql`.

## Если что-то не работает

- **«Не удалось открыть лобби: not owner or project missing»** — проект ещё не
  успел сохраниться в БД. Это уже исправлено в коде (запись досылается перед
  публикацией). Если всё равно видишь ошибку — убедись, что `setup.sql`
  выполнен и ты вошёл под своим Discord-аккаунтом.
- **Удалил проект, после перезапуска вернулся** — тоже исправлено: очередь
  записей теперь досылается на сервер при старте.
- **«function publish_project does not exist»** — `setup.sql` не выполнен,
  прогони его.

> Старые файлы `schema.sql` и `migrations/` оставлены для истории, но больше не
> нужны — используй `setup.sql`.
